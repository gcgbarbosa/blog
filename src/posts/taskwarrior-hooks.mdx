[Taskwarrior](http://taskwarrior.org/) is a CLI task manager that lets you track your to-dos using plain text.
It is great. Seriously.

You type the following to the CLI:

```bash
task add "do laundry"
```

Then if you do:

```sh
task list
```

It will be there:

```text
ID Active Age   Description   Urg
1          6s   do laundry    0
```

You can set the task as active by typing:

```sh
task start 1
```

## Writing hooks

Now, what if you want to trigger an action when you start a task?
Let's do that.
TaskWarrior hooks are scripts that run when certain events happen,
letting you extend TaskWarrior’s behavior.

As an example, we are going to show a notification on Ubuntu when a task is started or finished with `task done <id>`.
On Ubuntu, sending a notification should look like:

```sh
notify-send "TaskWarrior" "Your task finished"
```

We are going to assume your TaskWarrior database is at `$HOME/.task` and your config at `$HOME/.taskrc`.

To get started, create a `hooks` folder inside your `.task`.
The types of hooks you can create can be found in the [documentation](https://taskwarrior.org/docs/hooks/).
We are going to use the `on-modify` hook.

The second thing you need to do is add this to your `.taskrc` file:

```text
debug=1
debug.hooks=2
```

Once you do this, when you run TaskWarrior, you're going to see a bunch of debugging information.
That will help you understand what is going on.

The magic happens in between.
You can write hooks in any language you want, as long as they are executable.
We're choosing python for this example.

So how do we define the type of hook? By naming our file.
The hook should start with `<name-of-hook><whatever>`.
My hook is named `on-modify.message`.

Let's start by reading the two lines of input.
The first line is the state of the task berfore being modified,
while the second is the state after.

```python
#!/usr/bin/env python3
import sys
import json

if __name__ == "__main__":
    before = json.loads(sys.stdin.readline())
    after = json.loads(sys.stdin.readline())

    print(json.dumps(after))
```

The last line of our script prints the line modified, essentially making our hook a no-op.

To achieve what our example wants,
we are going to get the message from one of the input lines,
then pass it to subprocess to call `notify-send`.

```python
if "start" in before and "start" not in after:
  description =  before.get("description", "empty description")
  subprocess.call(["notify-send", "TaskWarrior", description])
```

That's it!
The final script should look like:

```python
#!/usr/bin/env python3
import sys
import subprocess
import json

if __name__ == "__main__":
    before = json.loads(sys.stdin.readline())
    after = json.loads(sys.stdin.readline())

    if "start" in before and "start" not in after:
      description =  before .get("description", "empty description")
      subprocess.call(["notify-send", "TaskWarrior", description])

    print(json.dumps(after))
```

Save it to your `hooks` folder.
Now when you run `task <id> start`, there should be a notification on your desktop.
